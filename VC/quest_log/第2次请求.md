# 第2次请求

**时间**: 2026-05-25

## 请求内容

### 第1轮
用户要求改为空间中随机生成几何体（正方体、球体等），无限生成，可以穿梭。

### 第2轮  
用户指出渲染不正确，要求真实的相对论光学效果，且要求视角控制运动方向。

## 最终实现方案

### 渲染管线：GPU光线步进
放弃 CubeCamera 方案，改为**每像素独立相对论光线步进**：

1. 屏幕像素 → 相机空间射线
2. 相机空间 → 观察者系方向（View Matrix）
3. **逆洛伦兹变换**（光行差公式）→ 静止系射线方向
4. **SDF光线步进**：静止系中追踪射线，对128个物体求交
5. **多普勒频移 + 强度变换**：命中物体后计算 ω_obs/ω_rest 和强度因子

### 物理正确性

逆光行差公式（静止系 → 观察者系）：
```
n̂ = [n̂' + (γ-1)(n̂'·β̂)β̂ + γβ⃗] / [γ(1 + β⃗·n̂')]
```

多普勒因子：
```
ω_rest/ω_obs = γ(1 + β⃗·n̂') = dopplerInv
```

强度因子：
```
I ∝ 1/dopplerInv² × (1 + β·3.5·max(0, -n̂'·β̂))
```

### 技术架构

| 组件 | 技术 |
|---|---|
| 渲染 | Three.js WebGL 2 + 自定义 ShaderMaterial |
| 场景几何体 | 128个SDF物体（球体+正方体），通过Float32 DataTexture传给着色器 |
| 世界生成 | 分块系统（160ls/块, 9³=729活跃块），确定性种子 |
| 控制 | W加速/减速，鼠标控制速度方向，点击锁定指针 |
| 两种模式 | 自由探索（W加速移动）/ 瞬间观察（冻结状态旋转观察） |

### 着色器技术细节

- GLSL ES 1.0 API兼容（`varying`, `texture2D`, `gl_FragColor`）
- 避免 `out` 参数（部分驱动不兼容），使用全局变量传递射线命中信息
- 128个物体 × 80步光线步进 = 每像素最多10240次SDF评估
- 球体SDF: `length(p) - r`
- 正方体SDF: `max(abs(p)-b)` + 混合

### 修复记录

- v0: CubeCamera + Three.js InstancedMesh → 高速时Cubemap分辨率不足
- v1: 直接全屏着色器光线步进 → GLSL兼容性修复（移除 `#version 300 es`、`out` 参数）
- 种子使用 `>>> 0` 确保无符号正数（兼容负分块坐标）
- 分块密度提高（半径 4 → 9³ 分块，每块 45 物体）
